# Support

Use GitHub Issues for reproducible bugs and focused feature requests.

Before opening an issue:

- Check the latest release.
- Search existing issues.
- Include the operating system, architecture and app version.
- Include logs only after removing tokens, usernames or machine-specific private paths.

For packaging or release problems, include the exact asset name you downloaded, for example `PulseRing_1.0.0_x64-setup.exe` or `PulseRing_1.0.0_aarch64.dmg`.
